## Key Statements
- The Dump folder removes cognitive friction entirely, enabling focus on original thought and value production rather than mechanical sorting
- Sparse indexing optimizes for discovery; Manifests optimize for lightweight answer retrieval -- different tools for different precision requirements
- The real difficulty is not building the system but upgrading/debugging it, especially context/token management
- FrontMatter schema design is where Polanyi's Paradox manifests concretely: Vincent cannot fully specify his own retrieval needs before trying
- AI-generated FrontMatter fields suffer two problems: information redundancy (fields never queried) and field omission (gaps hard to notice)
- Pending Terms folder evolved from simple naming decisions into a contradiction-parking mechanism that enables creative exploration of both sides of unresolved design tensions
- Deprecated content contamination is a real failure mode; semantic search (not vector) is the remedy because vocabulary already removed from the system can still be found by meaning
- All work happens in a single interface (Obsidian/VS Code), eliminating context-switching friction
- Three-tier enterprise KB model proposed: personal DB + initiative DB + company RAG, with vector embedding only at project start and end
