## Key Statements
- Dump folder as trust mechanism: zero-friction ingestion removes cognitive load, frees bandwidth for original thinking.
- Sparse indexing = navigation layer ("where to look"); Manifest = compression layer ("what's there"). Different tools for different precision requirements.
- Pending Terms folder evolved from simple naming decisions into a productive contradiction-parking architecture; unresolved forks can be developed in parallel until resolution emerges organically.
- FrontMatter schema cannot be fully specified upfront -- this is Polanyi's Paradox in action. Field design requires iterative use-and-prune.
- Tag List was AI-proposed; solves tag explosion by requiring reuse of existing tags before creating new ones.
- Deprecated content contamination is a real failure mode; semantic search outperforms vector search for finding conceptual remnants that have been removed from the formal ontology.
- System build speed is surprisingly fast (3rd/4th system: <10 min each via standardized System Spec). Debugging and upgrading is surprisingly hard.
- Context management as forced skill development; token cost as first-class architectural constraint.
- Enterprise three-tier model: personal KB (tacit knowledge) + initiative KB (project-scoped, file-based) + company RAG (vector-embedded at initiative close). RAG updates only at project start/end, not continuously.
