# Session c888ce5d-5009-4208-97e9-832a6ba17b13
## Topic
设计并实现 MeowOS dashboard-updater agent, 解决 Dashboard 每次全文重写的 token 浪费问题. 借鉴 Horsys 的 horsys-dashboard skill, 最终选择专用 agent 方案.

## Facts & Decisions
- MeowOS 此前没有专用 Dashboard 更新机制, 每次更新涉及读全文再重写
- 参考了 Horsys 系统的 `horsys-dashboard` skill (外科手术式更新, 只动被传入字段)
- 评估三方案: A. Claude Code Skill (仿 Horsys), B. 专用 Agent (90_Agents/), C. 增强 shell-runner 指令
- **决定采用方案 B**: 专用 `dashboard-updater` agent, 理由: token 最省 (主 session 不读 Dashboard 原文) + 格式知识集中 + 与现有 agent 体系一致
- 定义 8 个 action: update_timestamp, replace_schedule, update_schedule_status, upsert_近期日程, add_todo, complete_todo, sync_fitness, upsert_section
- sync_fitness 从 `_state.md` 派生, Dashboard 健身区块是派生视图而非 source of truth
- replace_schedule 保留 `> warning` 注释块的防护规则
- 跳过 spec 文档, 直接执行实现
- 修改了 6 个文件: 新增 dashboard-updater.md, 修改 01_Routing.md / dump-processor.md / scheduler.md / daily-review.md / fitness-advisor.md
