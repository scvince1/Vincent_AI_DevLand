Polanyi's core claim: "We know more than we can tell."

Humans hold knowledge that cannot be fully articulated in language. An experienced manager knows what situations warrant concern but cannot produce a complete list of criteria if asked.

Vincent's extension (from LinkedIn Article 1): Polanyi's formulation misses a second face. We also NEED more than we can describe. The gap between specification and actual requirement is structural, not a communication failure. Both faces together constitute Human Sloppiness.

Implications for AI system design:
- You cannot expect users to one-time define all their judgment criteria for the system
- The correct pattern: let the system generate judgments, let users react to specific cases ("yes" or "no, because..."), let the system calibrate from concrete feedback
- "Review and approve" mode is more effective than "configure from scratch" mode
- Turning the hard thing (defining rules abstractly) into the easy thing (reacting to specific proposals)
