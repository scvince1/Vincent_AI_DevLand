**重要诚实声明：本节是调研中最薄弱的部分。** Revuze 在所有官网页面均不公开具体图表类型或 UI 细节——这是刻意的营销策略（"See a Live Demo"作为核心 CTA 出现 4+ 次）。

### Reporting Hub 的 8 类报告（最具结构性的 UI 线索）

1. Product Launch Report — 产品表现、强弱项分析
2. Comparison Report — 品牌独特价值 + 未开发市场机会
3. Market Trends Report — 趋势跟踪，区分持久 vs. 昙花一现
4. Product Superiority Report — 产品优越性定义与展示
5. Purchase Motivation Report — 消费者评分驱动因素
6. Usage & Attitudes Report — 使用场景与时间分析
7. Brand Tracker Report — 声量/情绪/星级/产品趋势综合追踪
8. Market Penetration Report — 购买驱动、头部品牌、新兴趋势深度分析

**格式推断（非确认）：** 基于"Brand Tracker"包含 volume + sentiment + star rating + trends 四个维度，推测存在时序折线图或多维 dashboard；无法确认轴定义或图表类型。

### CI Hub 的视觉线索（最接近竞品图表描述）

CI Hub 页面提到：
- "Six Sets of VoC Data: Research and reviews, social listening, communities and forums, surveys, focus groups and customer care"
- 竞争性仪表板（competitive dashboards）概念存在，但**无轴定义、无散点图/气泡图的明确描述**

**对比 Vincent 的基准（Consumer Insights Hub = 2D 散点图，X 轴 = Volume / Y 轴 = Sentiment）：** Social Hub 在公开材料中**完全没有**等价的可视化规格描述。

## 4. Platform Coverage — 平台覆盖实际情况

### 官方明确声称的数据来源平台

| 平台 | 出现位置 | 上下文 |
|------|---------|-------|
| TikTok | Social Hub 主页 hero section | "Social, commerce, TikTok and more" — 以独立标签列出 |
| TikTok Shop | 独立产品 Hub | 仅商业交易数据（sales revenue / units sold / creator performance）；**非 TikTok 有机内容** |
| Amazon | FAQ / eComm Hub | 评论数据来源 |
| Walmart | FAQ | 评论数据来源 |
| Target | FAQ / eComm Hub | 评论数据来源 |
| Sephora | FAQ / eComm Hub | 评论数据来源 |
| "Social media" | Platform 页 / 多处 | 泛称，从不展开具体平台 |

### 明确缺席（全站未提及为数据来源）

| 平台 | 状态 | 备注 |
|------|------|------|
| **Reddit** | 完全未提及 | 全站搜索零命中（非数据源上下文） |
| **Twitter/X** | 仅出现在页脚社交链接 | 为 Revuze 自己的 Twitter 账号，**非数据采集来源** |
| **Instagram** | 未提及为数据源 | 同上，仅 footer 图标 |
| **Facebook** | 未提及为数据源 | 同上 |
| **YouTube** | 未提及为数据源 | 有"Video Analysis"功能但平台未点名 |
| **LinkedIn** | footer 链接 | Revuze 自有账号，非数据源 |

### 关键发现：TikTok 有机 vs. TikTok Shop 区分

Revuze 将 TikTok 一分为二：
- **TikTok Shop Hub（独立产品）：** 专注商业交易数据——产品销售额、Top 趋势品类、创作者带货 ROI。这是电商情报，**不是**有机社交聆听。
- **Social Hub 中的 TikTok 提及：** 仅在 hero section 的数据来源列表中以"TikTok"标签出现，无具体功能说明。**是否覆盖 TikTok 有机内容（非 Shop）无法从公开材料确认。**

## 5. Pre-purchase vs. Post-purchase — 信号类型判断

**结论：以前置（pre-purchase）信号挖掘为主，兼顾后置。**

| 方向 | 证据 |
|------|------|
| Pre-purchase 主导 | "Spot emerging trends early, before they impact demand or performance" |
| Pre-purchase 主导 | "Plan campaigns using live market signals, not assumptions" |
| Pre-purchase 主导 | "See new competitor launches entering the conversation" |
| Pre-purchase 主导 | Category Launches / Trending Hashtags 功能定位 |
| Post-purchase 辅助 | "Organic vs. Incentivized: Evaluate organic vs. incentivized feedback" |
| Post-purchase 辅助 | eCommerce Intelligence 整合评论数据（VoC review data） |

整体定位：**战略型趋势 + 竞争监测**，后置评论数据作为佐证层，不是核心。

## 6. 诚实评估：公开情报的局限性

这份调研受到以下结构性限制：

1. **UI 黑盒：** Revuze 有意将所有 UI 细节挡在 Demo 后面。截图文件名可见，内容不可见。没有任何公开材料描述具体图表类型。
2. **平台覆盖语焉不详：** 全站使用"social media"泛称，唯一被命名的社交数据来源是 TikTok（且有机/商业边界模糊）。Reddit、Twitter/X 在数据源上下文中**完全缺席**。
3. **第三方评测封锁：** G2（403）、Capterra（404）、TrustRadius（403）均无法访问，无法获得用户实证。
4. **宣称 vs. 实际的不对称：** "No more Boolean queries"这类说法暗示底层是 AI 语义理解而非关键词匹配，但具体 NLP 架构不可知。

## 7. 战略含义提示（升维设计参考）

基于上述情报，Revuze Social Hub 的架构盲点：

| Revuze 做不到的 | 公开证据 |
|----------------|---------|
| Reddit 社区原生信号挖掘 | 全站零提及 |
| Twitter/X 有机舆情（非营销内容） | 仅出现在 Revuze 自己的 footer 链接中 |
| TikTok 有机内容（非 Shop 交易数据）深度覆盖 | TikTok 提及仅在商业 Shop Hub，有机覆盖无证据 |
| UI 层面的可解释竞争地图（如 2D Sentiment × Volume 双坐标） | 公开材料中无任何图表规格描述 |
| 非结构化前置信号（论坛/社区/UGC）与电商数据的实时融合 | Product Hub / eComm Hub 数据孤岛化，无 real-time fusion 描述 |
