# Consumer Sentiment Dashboard — SharkNinja Challenge 5 v2

**Brandwatch, Sprout, Meltwater and Talkwalker all score "Oh great, another vacuum that dies after 3 months" as positive.** Our dashboard does not. That one sentence is why SharkNinja should not be paying six figures a year for generic social listening tools when the product they are listening to is a $600 appliance that people review the way they review a bad roommate — sarcastically, comparatively, aspect by aspect, and in the vocabulary of brushrolls, carafes, HEPA filters and descaling cycles. We built a consumer-electronics-aware NLP pipeline, wrapped it in a clean five-page dashboard, and kept the receipts: a pytest suite that proves on real sentences where VADER fails and where our pipeline wins.

---

## What this is

A decoupled FastAPI + React/TypeScript/Vite social listening dashboard MVP, purpose-built for SharkNinja's consumer intelligence use case. Five pages — Overview, Product Analysis, Platform Comparison, Topic Explorer, Alerts & Insights — each tied to a specific business question owned by a specific persona. The stack is intentionally boring; the NLP layer is intentionally not.

## Why this beats the incumbents

Generic social listening is built for brand reputation monitoring. It treats a SKU-level product review the same way it treats a press mention or a political tweet, which means:

- **Sarcasm is inverted.** "Wow, a $400 blender that can't crush ice. Revolutionary." scores positive on lexicon-based engines because "wow" and "revolutionary" are high-weight positive tokens. We catch it.
- **Comparative claims are averaged.** "Shark is way better than Dyson at edge cleaning" gets counted as one generic positive mention in most tools, with no attribution. We emit two aspect-brand pairs: Shark gets the credit, Dyson takes the hit, both tagged to the `edge_cleaning` aspect.
- **Aspects collapse to one number.** "Suction is incredible but the dustbin is tiny and the battery is garbage" becomes a single near-neutral score everywhere else. For us it is three aspect scores — suction positive, dustbin negative, battery negative — and that is the only view that actually helps a PM decide what to fix.
- **Domain vocabulary gets dropped.** Generic NLP treats "brushroll", "carafe", "descale", "agitator" as unknown tokens and lets them fall out of the analysis. Our domain lexicon (40+ terms covering vacuums, kitchen, coffee, air care, hair tools) makes them first-class entities.

Every claim above is backed by a test case in `backend/tests/test_nlp_edge_cases.py`. Sixteen cases, four per category, with the failing-before baseline (VADER alone) and the passing-after result (enhanced pipeline) on record. This is the evaluation criterion — we did not hand-wave it.

## Architecture at a glance

```
Challenge_5_v2/
├── backend/               FastAPI service — owned by backend-engineer
│   ├── app/
│   │   ├── nlp/           enhanced NLP pipeline + domain_lexicon.py
│   │   ├── scrapers/      BaseScraper ABC + CSVAdapter (MVP) + future drop-ins
│   │   └── routers/       depend on the ABC, never on CSVAdapter by name
│   ├── models/
│   │   └── schemas.py     Pydantic — single source of truth for API shapes
│   └── tests/
│       └── test_nlp_edge_cases.py   16+ cases, sarcasm / comparative / ABSA / domain
├── frontend/              React + TS + Vite — owned by frontend-engineer
│   └── src/
│       └── pages/         Overview / ProductAnalysis / PlatformComparison / TopicExplorer / Alerts
└── contracts/
    ├── requirements.md    Business contract (personas, REQ-IDs, edge cases, data model)
    ├── api-contract.yaml  Generated from schemas.py — the only file frontend reads for types
    └── review-round-*.md  Review findings from business-leader
```

Key architectural rules (enforced, not suggested):

- **Scraper is an ABC.** `BaseScraper` lives in `backend/app/scrapers/base.py`. `CSVAdapter` is the MVP implementation. Real platform scrapers (Reddit, Trustpilot, Amazon, YouTube) drop in later via FastAPI `Depends(...)` with zero router edits. If a router imports `CSVAdapter` by name, that is a bug.
- **Pydantic is the contract.** `backend/models/schemas.py` is the single source of truth for API request/response shapes. `contracts/api-contract.yaml` is an export snapshot of those schemas, generated by backend and committed. Frontend builds TypeScript types from the YAML. Frontend never reads `schemas.py`.
- **File ownership is hard.** `business-leader` owns `contracts/requirements.md`, `contracts/review-round-*.md`, and this README. `backend-engineer` owns everything under `backend/` plus `contracts/api-contract.yaml`. `frontend-engineer` owns everything under `frontend/`. Overwrites across ownership lines are a project-level failure.

## The five pages, the five questions

| Page | Question it answers | Primary persona |
|---|---|---|
| **Overview** | What is the state of consumer sentiment across our portfolio right now, and what changed this week? | Consumer Insights Manager |
| **Product Analysis** | For a specific SKU, what are people saying about each aspect of the product, and how is each aspect trending? | PM, Shark Robot Vacuums |
| **Platform Comparison** | How does the conversation about this product differ across Reddit vs Amazon vs YouTube vs Trustpilot, and are the aspects the same? | Marketing Ops |
| **Topic Explorer** | What aspects are people talking about that we didn't pre-define — and are any gaining or losing ground? | Competitive Intelligence Lead |
| **Alerts & Insights** | What should I be paying attention to right now that I don't already know about? | Director of CX |

Each page header shows the question as a subtitle. That is deliberate. A dashboard that cannot answer a named question in a named persona's voice is decoration.

## The NLP pipeline in one paragraph

VADER is the baseline, nothing more. On top of VADER we layer: (1) a sarcasm heuristic using contradictory polarity between framing tokens and content tokens plus conditional-clause detection, (2) comparative-sentiment extraction via dependency-parse patterns that split on `better/worse/slower/louder than` and emit one aspect-brand pair per side of the comparison, (3) an ABSA module built on aspect-term extraction plus per-aspect polarity classification so that "suction positive, dustbin negative, battery negative" comes out as three structured records rather than one averaged number, and (4) a domain lexicon at `backend/app/nlp/domain_lexicon.py` that protects consumer-electronics vocabulary through tokenization, POS-tagging and aspect normalization. The test suite exercises all four in isolation and in combination.

## Getting it running

```bash
# Backend
cd backend
python -m venv .venv && source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install -r requirements.txt
uvicorn app.main:app --reload --port 8000

# In a separate terminal, frontend
cd frontend
npm install
npm run dev    # opens http://localhost:5173

# Prove the NLP claim
cd backend
pytest tests/test_nlp_edge_cases.py -v
```

Exact commands are subject to backend-engineer and frontend-engineer plan approval — this README will be updated once their plans are approved and the scaffolds land. The pytest command above is not negotiable; that is the evaluation artifact.

## Demo script (suggested)

1. Open **Overview**, filter to Shark Robot Vacuums, last 30 days. Point at the week-over-week delta and the top rising negative aspect.
2. Drill to **Product Analysis** for PowerDetect UV Reveal. Show the aspect breakdown table — suction, navigation, mop pad, app, self-empty base — and point at the aspect that's trending down.
3. Click through to a sarcastic Reddit exemplar mention from the drill-through panel. Note that it is correctly scored negative.
4. Switch to **Platform Comparison** and show that Reddit and Amazon surface different top topics for the same product — this is invisible in Brandwatch's default view.
5. Open a terminal. Run `pytest backend/tests/test_nlp_edge_cases.py -v`. Watch 16+ tests go green. Close the laptop.

## Out of scope (on purpose)

MVP uses CSV fixture data, not live scraping — the `BaseScraper` ABC means live scrapers drop in later without architectural churn. Non-English mentions are flagged and excluded rather than mis-scored. Single-tenant, no auth. No image or video sentiment. No write-back to source platforms. These are not regrets, they are scoped decisions; `contracts/requirements.md` section 8 explains each.
