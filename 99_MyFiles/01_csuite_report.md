## 核心命题

> 这个系统真正做的事情，是**改变人与思维之间的接口**，而不只是改变人与工具之间的接口。它不是一套效率工具的集合，而是公司的认知基础设施——让每一个做判断的人，在做判断的那一刻，手里刚好有他需要的全部上下文。

---

## 一、我们在解决什么问题

我们公司目前面临的核心挑战，不是效率问题，而是一个**组织认知问题（Organizational Epistemic Problem）**：信息存在，但分布不可见；工作在推进，但状态不透明；决策在发生，但逻辑在消失。

### 三重对齐困境

| 对齐类型 | 英文 | 问题描述 |
|---|---|---|
| **信息对齐** | Aligning Information | 谁知道什么、信息在哪里 |
| **进度对齐** | Aligning Progress | 有多少 Initiative 在跑、谁在负责、状态如何 |
| **盲区对齐** | Aligning Ignorance | 谁不知道什么、哪里有无人察觉的认知差距 |

这三类对齐工作，目前在高管层占据约 **30% 的工作时间**，且大量以会议和邮件的形式反复发生。真正的问题不是频率——高层对齐的频率是合理的——而是每次对齐都从零重建上下文，心智成本极高，且成果无法积累。

---

## 二、系统愿景：四层架构

系统由四个层级构成，信息在层级间双向流动，所有 Agent 在后台运行，不污染任何主会话的上下文。

| 层级 / Agent | 核心职能 | 触发方式 |
|---|---|---|
| **个人主权层 L0** | 每人私有的 Personal Vault + Personal Shell。绝对隔离，不进入任何组织数据库 | 主动发起 |
| **感知层 L1** | 邮件 / 日历 / 文件 / 报告的持续监听与结构化捕获 | 定时 + 事件 |
| **记忆与理解层 L2** | Initiative 注册表、组织知识图谱、决策日志的维护 | 信息流入时 |
| **专项功能层 L3** | 会议、报告编排、财务、跨区域桥接、跨项目合成等 Agent | 事件触发 |
| **编排与治理层 L4** | 权限矩阵、触发路由、审计流、降级协议 | 全程后台 |

> **关键设计原则：** 所有 Agent 只准备和呈现，不替人决策，不替人发送。任何输出在人工确认前均为草稿。

---

## 三、对高管层工作的实际影响

系统的价值不只是节省时间，而是**改变工作的性质**。两个具体的转变：

**角色转变：从信息加工者变为信息裁判。**
信息的大部分处理在到达决策者之前已经完成。高管面对的是已被结构化、已被连接、已被标注重要性的信息，工作是判断这个处理是否正确，而不是自己做处理。

**决策质量：盲区变得可操作，而不只是可感知。**
系统会在进入关键会议或决策场景之前，主动标注参与者之间的认知差距——谁可能不了解哪个关键背景。这让「重新对齐上下文」的工作从临场消耗变为事前准备。

## 四、我们现在在哪里，下一步讨论什么

### 目前已投入运行的基础

- AI 邮件助手原型（Outlook 接入，个人知识库，草稿生成）
- 财务文件自动处理流程（收据 → Transaction Log → Financial Statement）
- 约 20 名 Power Users 具备自主部署和调试能力，约 30 名 Hobby Users 在日常工作中积极使用
- 公司与 Anthropic 的企业 API 协议已就绪，数据合规框架已确立

### 建议本次讨论聚焦的三个议题

1. **优先级排序：** 组织认知层（Initiative Registry、知识图谱）与工具层（邮件、会议、报告）的投入顺序如何定？
2. **治理边界共识：** 哪些领域需要写入系统级的禁止接触清单，哪些领域 AI 可以主动准备但需人工确认？
3. **Champion 网络：** 如何把现有的 20 名 Power Users 培养成各区域的内部推广节点，让系统在无行政命令的情况下自然扩散？
