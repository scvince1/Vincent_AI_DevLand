## 设计

### 架构

一个本地 Python MCP server（`outlook-calendar-mcp`），通过 `win32com.client` 与 Outlook 桌面端通信，挂进 Claude Code 的 MCP 配置。凌喵通过 MCP tools 直接操作日历。

### MCP Tools（4 个）

| Tool | 功能 |
|------|------|
| `list_events` | 查询指定日期范围内的日历事件（默认今天+明天） |
| `create_event` | 新建日历事件（标题、时间、地点、备注） |
| `update_event` | 修改已有事件（按 EntryID 定位，改任意字段） |
| `delete_event` | 删除事件 |

### 技术细节

- **语言**：Python + `pywin32`（`win32com.client`）
- **MCP 协议**：用 `mcp` Python SDK（stdio transport）
- **Outlook 未启动时**：COM 调用会自动启动 Outlook 进程，不需要额外处理
- **日历访问**：`MAPI.GetDefaultFolder(9)` = 默认日历文件夹
- **时区**：事件时间统一用美东时间（你当前时区）

**Q2：双向冲突**

完全不会。MCP server 是**无状态**的——它不维护任何本地副本，每次调用都是直接读写 Outlook 本身的数据。所以：

- 你在 Outlook 桌面端改了事件 → 凌喵下次查的时候自动看到最新的
- 凌喵改了事件 → 你在 Outlook 桌面端立刻能看到
- 没有同步、没有缓存、没有冲突的可能
