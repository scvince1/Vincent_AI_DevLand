# KPI 测量指南：如何在不增加工作负担的情况下衡量系统价值
## KPI Measurement Guide: Passive Metrics for Cognitive Infrastructure

**COO Office · 2026**

---

## 前言：测量的悖论

我们希望测量的东西——认知协调质量、决策信息完整性、组织知识的分布状态——恰恰是最难被主动测量的东西。

传统的测量方式（问卷调查、手动记录、定期汇报）在这里行不通，原因有三：

1. **大家没有时间**主动配合填写测量工具
2. **主动测量会改变被测量的行为**——人们知道自己在被观察时会表现不同
3. **你想测量的是系统运转的自然状态**，而不是人们在测量框架下的表现

解法是**被动测量（Passive Measurement）**：从系统已经在处理的数据里提取测量信号，不需要任何人额外做任何事。

---

## KPI 一：信息延迟（Information Latency）

**定义**：一件重要的事情发生，到所有需要知道这件事的人真正知道它，平均需要多长时间。

### 为什么重要

信息延迟是认知基础设施最直接的健康指标。当一个市场事件发生、一个内部决策被做出、一个项目遇到风险——这个信息从产生到到达所有需要它的人手里，中间有多少时间损耗，直接影响组织的响应速度。

### 如何被动测量

**方法一：邮件扩散分析**

邮件感知 Agent 已经在处理所有邮件流。在这个基础上，追踪：

- 一封被判断为「信息密度高」的邮件（比如项目风险警报、市场异常通知），从发件人发出，到所有被标注为「相关受众」的人打开，平均经过了多少小时
- 「相关受众」的判断基于 Initiative Registry 的项目负责人列表和 Organizational Knowledge Graph 的连接关系

> **数据来源**：邮件感知 Agent 的已读状态追踪 + Initiative Registry 的受众映射
>
> **输出频率**：每周自动生成一次，无需人工操作
>
> **基准建立方式**：系统运行前三个月为建立基准期，记录各类信息的自然传播速度，之后以此为基准检测改善情况

**方法二：会议补课频率追踪**

在开启了转写功能的正式会议里，Natural Language Processing 检测特定语言模式的出现频率：「等等，这件事我不知道」「什么时候决定的」「我没有收到这个消息」。

这些短语的出现代表一次「信息延迟导致的临时补课事件」。追踪其在所有被转写会议里的出现频率。

> **数据来源**：会议转写文本的 NLP 分析
>
> **注意**：只在已开启录制的正式会议里运行，不适用于敏感或非正式会议
>
> **输出**：每月一次「信息补课事件」趋势报告

### 目标方向

不设置绝对目标数字（不同类型信息的合理传播时间差异很大），而是追踪**相对趋势**：系统部署后，信息延迟是否在持续下降？

## KPI 二：认知对齐失败率（Epistemic Alignment Failure Rate）

**定义**：在一段时间里，有多少次会议或决策场景里，因为某个参与者缺乏关键背景而需要临时中断和补课。

### 为什么重要

这个指标直接测量的是盲区对齐（Aligning Ignorance）工作的效果。每一次「临时补课」都代表了一次本可以被预防的认知失同步。

### 如何被动测量

**方法一：会前简报使用率 + 反馈信号**

Personal Shell 在每次会议前生成会前简报。追踪：

- 简报的打开率（多少比例的参会者在会前打开了简报）
- 简报的「有用」标注率（如果在简报里有一个简单的「有帮助 / 没帮助」按钮，追踪比例）
- 会议结束后，是否有与简报内容相关的后续邮件（比如「你在简报里提到的那个背景能详细说说吗」代表简报有效，「等等这个我完全不知道」代表简报漏掉了关键内容）

> **数据来源**：Personal Shell 的使用日志 + 会后邮件线程分析
>
> **特别说明**：「有用 / 没帮助」按钮是这个系统里极少数需要用户主动操作的测量点，但它的摩擦极低（一次点击），且直接服务于系统改进，值得要求

**方法二：会议节奏分析**

被转写的正式会议里，分析会议实际进行的时间分布：

- 用于背景介绍和上下文对齐的时间占比
- 用于实质性讨论和决策的时间占比

这个比例随时间的变化，反映了认知对齐效率的改善情况。

> **理想趋势**：背景介绍时间占比下降，实质讨论时间占比上升

### 目标方向

在系统部署前12个月内，将「背景介绍时间」在正式会议里的平均占比从基准值降低20%以上。

## KPI 三：决策捕获率（Decision Capture Rate）

**定义**：在所有被认定为重要决策的事件里，有多少比例在 Decision Log 里有完整记录。

### 为什么重要

决策日志是整个系统长期记忆的核心。捕获率低意味着组织的历史决策逻辑在持续漏水——那些「为什么当初这么决定」的答案随着时间消失，每次面对类似问题都需要重新推导。

### 如何被动测量

**方法一：决策信号自动识别**

会议 Agent 和邮件 Agent 被训练识别「决策信号」——语言模式表明一个决定刚被做出：

- 「我们决定……」
- 「最终方案是……」
- 「接下来我们会……因为……」
- 「这个问题，我们的结论是……」

识别到决策信号后，系统自动生成一个 Decision Log 草稿条目，包含：决策内容、参与者、时间、信息来源。发起人收到一个「这是我识别到的决策，请确认或补充」的请求。

决策捕获率 = 实际被确认进入 Log 的条目数 / 被识别到的决策信号总数

> **数据来源**：会议转写 + 邮件文本的 NLP 分析 + 用户确认率
>
> **注意**：这个 KPI 有一个重要的测量误差来源——系统可能漏掉一些决策信号（假阴性），也可能把非决策内容误判为决策（假阳性）。定期校准 NLP 模型的识别准确率是必要的

**方法二：历史查询需求追踪**

追踪用户向 Personal Shell 发起历史决策查询的频率和满足率：

- 发起了多少次「我们之前关于 X 是怎么决定的」类型的查询
- 系统能够从 Decision Log 里找到答案的比例

满足率低说明捕获率不够高，或者已有记录的结构化程度不够。

### 目标基准

Decision Log 捕获率目标：**前6个月达到60%，12个月内达到80%**。

这个目标是分阶段的，因为系统需要时间学习这家公司特定的决策语言模式。

## KPI 四：Initiative Registry 新鲜度（Registry Freshness）

**定义**：注册表里的条目，平均「最后更新时间」是多久之前。特别追踪超过4周没有任何信号更新的条目占比。

### 为什么重要

Registry Freshness 是整个系统的「信息鲜度指标」。注册表里的数据越陈旧，系统基于它生成的洞察就越不可靠。一张充满过时数据的注册表比没有注册表更危险，因为它会给出错误的确定感。

### 如何被动测量

这是四个 KPI 里**完全可以被自动测量**的一个，不需要任何人工干预。

系统持续维护每个注册条目的时间戳：

- 最后一次相关邮件提及时间
- 最后一次相关会议召开时间
- 最后一次负责人确认状态时间

每周自动生成一份健康度报告：

```
Initiative Registry 健康度报告 — 第X周
---
总条目数：[N]
🟢 绿色（14天内有更新）：[N] 条，占 [X]%
🟡 黄色（15-30天无更新）：[N] 条，占 [X]%
🔴 红色（30天以上无更新）：[N] 条，占 [X]%

需要关注：
- [项目名称]：[最后更新时间]，[负责人]，建议[动作]
```

### 自动触发的跟进动作

当一个条目变为黄色状态时，系统自动向负责人发送一条极短的更新请求：

*「[项目名称] 已经 [X] 天没有状态更新了。请用一句话告诉我：这个项目现在是正常推进、暂停、还是已完成？」*

这个请求的设计原则：响应成本极低（一句话），不是让人写报告。

### 目标基准

**绿色条目占比保持在 70% 以上**，红色条目不超过 10%。

## 综合健康仪表板

以上四个 KPI 可以被整合到一个每周自动生成的系统健康仪表板里：

```
组织认知基础设施 — 周健康报告
生成时间：[自动]

一、信息延迟
本周平均信息传播时间：[X] 小时（较上周变化：±[X]%）
高优先级信息平均延迟：[X] 小时

二、认知对齐
本周会议补课事件：[N] 次（较上月基准：±[X]%）
会前简报使用率：[X]%

三、决策捕获
本周识别决策信号：[N] 个
确认进入 Log：[N] 个，捕获率 [X]%
累计 Decision Log 条目：[N]

四、注册表新鲜度
🟢 [X]%  🟡 [X]%  🔴 [X]%
本周需要跟进：[N] 个条目

---
系统建议关注：[自动生成的1-3条异常信号]
```

这份报告不需要任何人手动准备，完全由系统自动生成，每周发送到 COO Office 和相关负责人。

---

## 一个重要的提醒：数字不是终点

这四个 KPI 是系统健康的代理指标（Proxy Metrics），不是目的本身。当你在追踪这些数字的时候，真正问的问题始终是：**「这家公司的决策质量有没有在提升？」**

这个问题本身是无法被直接测量的。但当信息延迟在下降、认知对齐失败在减少、决策历史在被完整记录、项目状态在被实时维护的时候，我们有理由相信答案是肯定的。
